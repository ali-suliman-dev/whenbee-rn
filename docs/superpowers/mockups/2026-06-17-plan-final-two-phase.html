The wheel (answer to "is it possible"): yes. We reuse the same scroll-wheel from Add Task — drag/flick up & down, center row is the selection (pill highlight), neighbours fade. Two uses here:
• Finish-by = a compact HH:MM wheel (two slim columns), narrower than the Add-Task duration wheel, with mode chips (leave by · be done by · be at) above. Native iOS feel; implementable with the existing custom wheel or a native picker.
• Build card duration = a slim wheel pinned far-right (30 / 35m / 40, 5-min steps). No play button in Build — running starts only in Phase 2.

Run rail: done = green node + check + strike-through, now = purple pulsing node, pinned (🔒, no handle), upcoming = hollow ring + drag + ▶. ▶ / "open timer" = the real stop-&-log timer (feeds calibration + honey + reclaim). Falling behind → Re-plan → amber "Should I trim?" cut card you approve. Never auto. Amber-never-red; red only on Abandon.
