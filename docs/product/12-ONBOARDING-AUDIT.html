Seven screens, mapped from source and captured on the simulator. The writing has voice, the reveal is genuinely good, the motion is mostly disciplined. And the flow collects four answers, shows you a personalized multiplier, then quietly uses a population average anyway.

✅ Implemented — branch worktree-feat+onboarding-wiring · 2026-07-16

❌ Not production-ready — 3 blockers, all silent

None of them throw. None of them show an error. A user hits all three and just… never gets a personalized honest number. That's the worst failure mode available for a product whose entire wedge is calibration.

The reveal computes m0, prints "2.4×", and says "Now you know by how much." The engine never reads it. archetypeSeed has exactly two readers in the whole codebase — both the Patterns tab's card label.

Identical. The quiz result is discarded by the engine and shown back only as a label. That's not a missing feature — it's the reveal making a claim the app doesn't honor.

// src/engine/priors.ts — new
/** Population prior, personalized by the quiz seed while the category is cold.
 *  m0 is a whole-person read; the category prior carries the shape. Blend, don't replace. */
export function seededPriorFor(categoryId: string, m0: number | undefined): number {
  const prior = priorFor(categoryId);
  if (m0 === undefined) return prior;           // skip path → today's behavior exactly
  return clampRatio(prior * (m0 / POPULATION_MEAN_M));
}

🔴 Blocker 3

"Skip to my type" bricks calibration — permanently, silently

The button labelled "Skip to my type" is the one path that never shows the type. It also skips categories — and that's what does the damage.

Not a hard block. Tasks and timers work — the picker unions a hardcoded seed six. applyLog writes to the DB correctly. But every UI read iterates tracked, so:

  • honest number never reflects the user's own logs
  • honeycomb: empty
  • Patterns: empty
  • no error, no empty state — they log into a void

Only escape: manually add a category in Settings. Nothing tells them to.

Fix — skip is deleted; quiz is mandatory (founder call)
quiz/0 — no skip affordance
↓ every step answered
reveal → categories — picked ≥ 1 enforced
↓ complete() — floor, not a gate
categories.length ≥ 1 — always
↓ hydrate finds rows
statsByCategory learns from log #1

Deletes the blocker at the root rather than routing around it. The old skip was rational precisely because the quiz did nothing — once the answers drive the number, skipping only costs the user their own accuracy. 4 taps, no typing, no account.

The floor stays as belt-and-braces: nothing should reach complete() empty now, but an empty tracked list is unrecoverable, so the invariant gets a hard guarantee instead of a routing assumption. categoriesStore.ts:22-24 already documents "the app needs at least one".

Watch after ship: quiz_started → quiz_completed. If a bounce appears, revisit.

🟠 Finding 4

The disabled CTA is unreadable — and doesn't look disabled

By deliberate design (AppButton.tsx:171-175) the pill face stays 100% opaque when disabled; only the label dims to opacity.disabled = 0.4. Because onIndigo is a dark ink, fading it toward bright indigo makes it sink into the fill instead of greying out. Dark mode is the worst case — and dark mode is what ships.

The rest, briefly

🟠 Finding 5 — your funnel is lying

No in-flight guard on any CTA. router.push doesn't dedupe → double-tap stacks screens. ready.tsx fires onboarding_completed twice. Back-swiping reveal→quiz→reveal re-fires quiz_completed + reveal_shown every round-trip.

Any activation baseline measured today is inflated. Fix before instrumenting decisions on it.

quiz/2: "Where does time run away from you most?" → Meetings · Chores · Errands · Deep work

categories: "Where does time slip most?" → Getting ready · Cleaning · Admin & email · Errands · Cooking · Out the door

Two screens apart. Only "Errands" overlaps. No carry-over. Say "Deep work" and it isn't even offered as a category.

🟡 Finding 10 — clips at large type

No screen scrolls. QuizOption.tsx:58 uses fixed height (104/64pt) and the coin-edge hardcodes the same H — fix it in one place and the edge detaches. At AX sizes: spacer collapses, then content clips, then the CTA is squeezed off. No scroll = unrecoverable.
