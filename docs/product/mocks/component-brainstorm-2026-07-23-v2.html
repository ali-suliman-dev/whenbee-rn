What changed structurally / risks:
  • No card, no pills — a floating tray with round 46px icon buttons. The float + circles = reaction-picker language, the most universally tappable-looking UI that exists.
  • Glyphs at 26px on the tray circles — same animation-stage win as tiles.
  • Risks: two trays stacked is a lot of float; labels at 11px are small (accessibility check needed); playful register may read lighter than "honest number" wants.
Recommendation — 1·II Glyph tiles, with 1·I's collapse-to-receipt as an optional layer later. Tiles are the biggest visual break from "illustration-looking pills": a board of floating dishes reads as a game you play, the glyph animations become the hero at 26px on pure white, and selection (wash + ring + badge) never touches the glyph — so the draw animations survive every state, the flaw that killed 1C. 1·I wins on completion psychology (one question, radios, flavor hints) but costs sequencing logic; if tiles feel too static on device, graft 1·I's receipt-collapse on top. 1·III is the wild card — great energy, but two floating trays fight the honey card for depth.

Visibility rule (as requested):
  • Recap renders ONLY when: no active timer AND no unfinished planned task for today AND ≥1 completed log today.
  • Timer starts or a task is added → recap unmounts (plain unmount, no exit animation — Fabric rule). Task finishes / list empties again → recap fades back in with updated numbers.
  • Zero logs + zero plan (fresh morning) → recap does NOT show; that state keeps the existing first-run/plan empty treatment.
