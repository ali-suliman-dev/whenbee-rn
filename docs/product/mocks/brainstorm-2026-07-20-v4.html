v4 — both ends settable

D1's two-row chooser is locked. The open question is the start row: it used to mean only "as soon as I can". Now it can carry a real time, so the control becomes symmetric — pick which end you're fixing, and set it.

One copy trap worth naming before you choose. The word changes meaning depending on who set it. When you pick 09:30, the day is built forward from there — you start at 09:30. When Whenbee derives it from your finish time, it's the latest moment you can begin — you start by 11:00. Same clock, different promise. Every option below keeps that split: a number you set says at, a number Whenbee worked out says by. Collapsing both to "start by" would quietly turn a plan into a deadline.

Two identical rows. Two labels, two times, two derived clocks. Nothing about the start row is special-cased — whatever you can do to one end you can do to the other, which is the whole point of the merge.

"Now" isn't a separate concept here, it's just the default value: the row reads Now › until you pick something, and the picker's first entry is "Now". Tapping either time selects that row and opens the picker in one gesture.

Costs one tap to get back to "now" once you've set a time — you reopen the picker rather than hitting a chip.

The edge case that decides whether this feels honest. You set 09:30 this morning; it's now 14:15. The row keeps your 09:30 (it's your intent, not an error) and states plainly what's actually happening underneath — no red, no "you missed it", no auto-rewriting your number behind your back.

Row labels

SetStart rowFinish rowRead
A · symmetric verbsStart at · Now ›Finish by · 15:30 ›Two words each, perfectly parallel, and the at/by split does the semantic work for free. My pick.
B · your wordsStart byEnd byNicely parallel, but "start by" is the phrase the app already uses for the derived latest-start — reusing it for a value you set makes the two meanings collide.
C · first-personI'll start atI need to be done byWarmest, and unmistakable about intent. Second label is long enough to wrap next to a time pill on a small phone.
D · bareStartFinishCleanest column of all. Loses the at/by distinction entirely, so a set time and a derived time look identical.

On the sheet title: I'd keep "Today's plan". It names the object, not the mechanism — and now that both ends are settable, any title that mentions a start or a finish would be describing only half the control.
